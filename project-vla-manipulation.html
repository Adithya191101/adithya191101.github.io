Project Overview

This project implements Vision-Language-Action (VLA) models for robotic manipulation on a low-cost SO-101 robot arm. Two architectures were explored: ACT (Action Chunking Transformer) achieving 80% success, and SmolVLA with 9 iterative versions including systematic hyperparameter optimization using Optuna (30 trials). Comprehensive action logging and diagnostic analysis identified gripper oscillation as the core challenge in SmolVLA fine-grained control.

Model Architectures

Two complementary approaches to vision-based robotic manipulation, each with distinct strengths.

System Pipeline

The inference pipeline uses Real-Time Chunking (RTC) for asynchronous action generation while the robot executes.

Dual Cameras (640×480 @ 30fps) → Image Preprocessing + Augmentation → Vision Encoder (Frozen) → Language Instruction Embedding → Action Expert (Flow Matching) → RTC Action Queue → SO-101 Motor Commands

Gripper Oscillation Diagnosis

All 9 SmolVLA models with vastly different hyperparameters showed identical behavior: the robot approaches correctly but gets jittery near the grasp point. Added action logging during inference to capture every prediction — revealing the root cause.

Finding: Gripper Oscillates Between Open and Closed

t=0.0s   gripper=0.48   (open)
t=5.6s   gripper=31.04 (closed)
t=11.8s gripper=2.29   (open!)
t=18.1s gripper=36.53 (closed)
t=31.3s gripper=-0.11 (open!)
t=37.7s gripper=32.78 (closed)

Max step-to-step gripper change: 30.1 degrees in a single step.

Root Cause: Training Data Imbalance

  • 76% of frames: Gripper open (approach phase) — normalizes to -0.54 sigma
  • 12% of frames: Gripper grasping — normalizes to +2.21 sigma
  • Model loss dominated by approach behavior; grasping requires a rare high-confidence prediction
  • RTC prefix guidance amplifies uncertainty between consecutive chunks

Critical Debugging Discoveries

Eight major challenges identified and systematically resolved over 3 months of iterative development.

Issues Solved

1. Language Instruction Mismatch
VLA models create different embeddings for different text. Using the exact training task string fixed 0% to 33% success.
2. Camera Configuration Swap
Physical cameras were reversed, causing complete spatial reasoning failure. Built preview tool to verify camera feeds against training data.
3. Overfitting (15.87 epochs)
Optimized to 6.5 epochs using HPO — 2.5x faster training with better generalization.
4. Robot Calibration Drift
Joint offsets drift over time. Must recalibrate before every evaluation session.
5. Camera Angle Shift
Camera physically moved since data collection. Built live preview tool to compare and realign with training data.
6. Starting State Inconsistency
Robot must reset to exact home position between episodes. Gripper state (open vs closed) is critical.
7. Config Compatibility
Model trained on newer LeRobot (Colab) had unsupported config fields locally. Removed fields from Hub config.
8. Gripper Oscillation
Diagnosed via action logging: 76% open-gripper training data causes model to oscillate. Same behavior across all 9 model versions confirms this is data-related, not hyperparameter-related.

Key Lessons Learned

VLA Deployment Insights

  • Exact Configuration Matching: VLA models require identical language, cameras, and calibration between training and deployment
  • Hyperparameters Have Limits: When 9 versions with different configs show identical behavior, the issue is data or architecture
  • Action Logging is Essential: Logging raw predictions during inference reveals issues that qualitative observation cannot
  • Data Distribution Matters: Imbalanced action distributions (76% open gripper) cause the model to hedge on underrepresented behaviors
  • Systematic HPO Pays Off: Optuna found LR 2.4x lower than manual tuning — but confirmed the core issue was not hyperparameters

Next Steps

  • Immediate: Test SmolVLA with n_action_steps=1 (re-observe every step to reduce gripper oscillation)
  • Short-term: Train Pi0.5 (~3B params) with LoRA fine-tuning on the same dataset for better grasp precision
  • Medium-term: Multi-object scenarios, diverse language instructions
  • Long-term: Multi-task VLA, open-source contributions to LeRobot
